## Common Rejection Reasons

### iOS

1. **Missing privacy policy** - Must be publicly accessible URL
2. **Insufficient app description** - Explain what app does clearly
3. **Poor quality screenshots** - Must show actual app functionality
4. **Bugs or crashes** - Test thoroughly before submission
5. **Missing required metadata** - Fill all required fields
6. **Misleading app info** - Screenshots and description must match app
7. **Incomplete functionality** - All advertised features must work

### Android

1. **Privacy policy issues** - Must disclose data collection
2. **Permissions not justified** - Explain why each permission needed
3. **Copyrighted content** - Ensure you own or license all content
4. **Misleading functionality** - App must do what description says
5. **Security vulnerabilities** - Keep dependencies updated
6. **Target SDK version too old** - Must target recent Android version
7. **Crash on launch** - Test on multiple devices/Android versions

## Notes

**Approval Times:**
- iOS: Usually 24-48 hours (can be 1-7 days)
- Android: Usually 1-7 days (can be longer for first submission)

**Expedited Review (iOS):**
- Available for critical bug fixes
- Request in App Store Connect
- Explain why urgent
- Usually approved within 24 hours

**Staged Rollout (Android):**
- Can release to 5%, 10%, 20%, 50% of users
- Monitor for crashes/issues
- Increase rollout percentage gradually
- Useful for first release
